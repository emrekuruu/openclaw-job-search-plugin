# Reactive Resume Notes

Use this reference when resume output should be shaped for Reactive Resume rather than only presented as plain CV text.

Repository:
- https://github.com/amruthpillai/reactive-resume

## Content-first rule

Optimize the content first. Reactive Resume is the presentation/editor layer.

## Recommended mapping

Structure output into:
- Basics / contact info
- Headline or target title
- Summary / objective
- Work experience
- Education
- Skills
- Projects
- Certifications
- Languages
- Links / profiles

## Tailoring heuristics

- Mirror listing phrasing only when the profile supports it
- Put exact-match keywords in high-signal zones:
  - title/headline
  - summary
  - relevant experience bullets
  - skills section
- Keep weaker history shorter
- Prefer achievement bullets over responsibility bullets

## Safe optimization rules

- Rephrase aggressively; do not fabricate facts
- Add metrics only when present in the source profile
- If the candidate lacks a required tool, emphasize adjacent tools and transferable outcomes
- If the listing is broad, keep the resume coherent instead of chasing every keyword

## Suggested delivery modes

### Resume draft
A polished human-readable tailored CV.

### Structured payload
A sectioned payload ready for manual entry or later transformation.

### Tailoring notes
Short explanation of:
- keywords targeted
- content emphasized
- content omitted or deprioritized
- unsupported requirements left unclaimed
