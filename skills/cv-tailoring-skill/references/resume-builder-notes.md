# Resume Builder Notes

These notes borrow useful patterns from the external `resume-builder` skill at ClawHub without expanding this skill beyond its project scope.

Useful takeaways:
- Never hallucinate missing profile details.
- Ask for clarification when core candidate facts are missing.
- Keep resume language concise and concrete.
- Prefer role-relevant ordering of sections and bullets.
- Tailoring is different from full resume generation: this skill should adapt existing evidence, not create a full resume format or Reactive Resume JSON payload.

Do not import the full external skill structure here unless the project explicitly needs Reactive Resume output.
