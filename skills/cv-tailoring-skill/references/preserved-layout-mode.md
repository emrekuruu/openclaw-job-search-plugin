# Preserved-Layout CV Tailoring

Use this mode when the user provides an existing CV and wants the tailored result to stay visually and structurally close to that base document.

## Rules

- Preserve section order.
- Preserve section titles.
- Preserve the overall compactness and density of the CV.
- Preserve the user's existing formatting style where practical.
- Treat the base CV as the source of layout truth.
- Treat profile/additional-info files as the source of factual truth for possible substitutions.
- Never invent new experience, projects, tools, dates, metrics, or organizations.

## Allowed edits

- Reorder skills within the existing skills section.
- Rewrite bullets lightly to better match the target job description.
- Reorder projects by relevance.
- Replace a lower-relevance project with a higher-relevance verified project from additional info.
- Make small wording changes to improve recruiter readability.

## Disallowed edits

- Do not create a new resume layout.
- Do not add or remove major sections unless the user explicitly asks.
- Do not turn a compact CV into a long-form resume.
- Do not overfit every bullet to the job posting.

## Suggested workflow

1. Read the base CV first and infer the template.
2. Read the target job description.
3. Read verified additional info only if present.
4. Lock the section order and titles.
5. Tailor wording and ordering within those constraints.
6. Produce markdown or HTML that keeps the same overall shape.
